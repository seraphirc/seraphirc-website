Why SeraphIRC exists

IRC is not dead. It is under-served.

IRC still does something rare: it gives communities fast conversation without a mandatory corporate identity layer. Open-source rooms, long-running networks, small friend servers, and self-hosted bouncers keep proving the protocol still matters.

SeraphIRC is for people who want that world to feel good on a modern desktop or phone without sanding away the things that make IRC useful: real networks, slash commands, channel culture, services, bouncers, and protocol details you can inspect.

It is calm by design, technical when it needs to be, and honest about being alpha software.

Built for real IRC use

IRCv3 compatibility

IRCv3 capability support

SeraphIRC negotiates modern IRC capabilities where networks and bouncers advertise them, while preserving classic IRC behavior as the baseline. This table reflects alpha behavior: supported pieces are usable, partial pieces are still being broadened, and coverage may change as the client stabilizes.

Supported Partial
Capability Status Alpha behavior
sasl Supported Requests SASL when enabled and advertised; the PLAIN authentication flow is implemented.
server-time Supported Parses the @time tag and uses valid server timestamps on messages.
message-tags Supported Parses and unescapes generic IRCv3 message tags.
cap-notify Supported Handles CAP NEW and CAP DEL after the initial handshake.
account-notify Supported Tracks account login/logout events and updates channel state.
account-tag Supported Reads the @account tag on messages when present.
extended-join Supported Parses the account and real-name fields sent on JOIN.
away-notify Supported Tracks away and back changes while the capability is active.
chghost Supported Applies ident/host changes and updates known user state.
multi-prefix Supported Parses stacked membership prefixes such as @+nick.
invite-notify Supported Surfaces invite notifications without auto-joining channels.
echo-message Supported Suppresses duplicate local echo and relies on the server echo.
batch Supported Tracks batch start/end and preserves batch references in message metadata.
msgid Supported Parses the msgid message tag and keeps it in message metadata.
draft/chathistory Supported Requests and replays missed or past messages from a server or bouncer (chathistory batch type).
draft/read-marker Supported Shares the server-authoritative read position via MARKREAD, so unread state stays in sync across desktop and mobile and survives relaunch.
monitor Supported Uses MONITOR to track online/offline presence for a watch list of nicknames.
draft/no-implicit-names Supported Negotiated so the server skips the implicit NAMES burst after join; SeraphIRC requests names when it needs them.
soju.im/bouncer-networks Supported Used for native Soju mode: discover, attach to, and manage bouncer networks from one connection.
labeled-response Partial Used for labeled WHOIS/NAMES and bouncer attach; other response flows use fallback correlation.
setname Partial Negotiated where advertised; real-name (SETNAME) changes are tracked, with fuller in-client editing still in progress.
standard-replies Partial Negotiated where advertised; FAIL/WARN/NOTE replies are accepted, with richer surfacing still being broadened.
Persistent IRCv3 metadata Partial Tag and msgid metadata is preserved in memory; on-disk persistence is not required yet.

Statuses reflect current alpha behavior. Capabilities are negotiated only where the network or bouncer advertises them, and coverage may broaden as the client stabilizes.

Quick start

Quick start guide

The first-run setup keeps direct IRC, ZNC-style bouncers, and Soju imports visible from the start. Use it to configure credentials, choose connection behavior, and get onto a real network quickly.

1. Start the Quick Setup

Upon first launch, you'll be greeted by the Quick Start Wizard. Click "Start Quick Setup" to continue, or Exit Initial Setup if you prefer to set things up on your own.

2. Choose a Network Source

After clicking next, you'll be prompted for a default provided network or a custom network not on the list. Advanced users leveraging Soju can choose "Import from Soju" to connect to their Soju bouncer and import their existing networks into SeraphIRC.

3. Pick a Connection Type

On the next screen, you'll be asked if you'd like to connect directly or use a ZNC/other type of bouncer. Note if you are using Soju, the previous screen will take you through the workflow for importing your networks.

4. Configure Network Details

On the next screen you can configure standard IRC network details such as nickname, alternate nick, SASL/NickServ, or autojoin channels.

5. Initialize Secure Storage

SeraphIRC uses your operating system credential store to securely store passwords. If you have not initialized your secrets store, you'll see a "Secure Storage Required" warning. Click "Retry Secure Storage" and you will be prompted for a credential store password. Once this is completed, click "Next" on the network details dialog.

6. Set Connection Behavior

On the next screen, you can choose whether your network connects at startup, auto reconnects, and what the reconnect delay is.

7. Activate the Network

Once you've clicked next, you'll be presented with a network configuration summary. When you're ready, click "Activate Network."

8. Finish Setup

Once your network is activated, you'll be able to Add Another Network or Finish Setup. Once setup is finished, your newly configured network will show up in your left treebar, and you're ready to go.
